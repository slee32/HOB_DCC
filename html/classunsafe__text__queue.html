Detailed Description

template<size_t size>
class unsafe_text_queue< size >

A simple, non-thread-safe queue for formatting text within one task.

This class uses the emstream structure and its "<<" operator to convert data to characters and write the characters into a simple circular buffer. This queue is not thread safe and must not be used for communication between tasks; it may only be used to buffer text which is used within the same task in which it is created. The syntax used for writing information to this buffer uses the overloaded shift operator "<<" in the same style as the C++ standard library's iostream objects such as cout.

Usage

In some task we create a pointer to a unsafe_text_queue object and use the new operator to create the queue itself:

unsafe_text_queue* p_my_queue;
...
main ()
{
...
p_my_queue = new unsafe_text_queue (50);
...
}

Data is put into the queue in the example below. Here, we use the Program Memory String macro PMS() to tell the compiler to read a string constant directly from program memory in order to save RAM; for information about PMS() see documentation for emstream.h:

uint16_t a_data_item;
...
*p_my_queue << PMS ("The data is: ") << a_data_item << endl;

Data can then read from the queue and then written to some serial device. In this case, the data will be written to an SD card, assuming that an object called my_card of class sd_card has previously been created. Note that we would not actually do this because the class sd_card is already a descendent of class emstream, so and SD card driver can format things with the << operator and doesn't need that kind of help. It's just an example:

if (p_my_queue->check_for_char ())
{
my_card->putchar (p_my_queue->getchar ());
}

This constructor creates a unsafe_text_queue object. This object can be used to make up character strings inside one task. It is not thread safe, so it must never be used to communicate things between tasks. The queue size should be large enough so that if the writing task gets ahead of the reading task for a little while, there's room in the queue to hold the extra characters; if not, writing to the queue will block until the reading task has read characters from the queue and done something with them.

Parameters
p_ser_devA pointer which points to a serial device which can be used for diagnostic logging or printing
a_wait_timeHow long, in RTOS ticks, to wait for a full queue to become empty before a character can be sent. The default value of portMAX_DELAY causes a send to block indefinitely

template<size_t size>
bool unsafe_text_queue< size >::putchar ( char  a_char)
virtual

This method writes one character to the queue. If the second constructor parameter wasn't given, the write operation will block until there is room in the queue for the character being written. Otherwise, the write will block for the given number of RTOS ticks waiting for an empty space in the queue, then if the queue has not become empty, give up in frustration and return false.

Parameters
a_charThe character to be sent to the queue
Returns
True if the character was successfully sent, false if something went wrong
